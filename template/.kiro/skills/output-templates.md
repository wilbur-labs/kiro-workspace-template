# Output Templates

## Commit Message (English)

```
<type>(<scope>): <subject>

<body>
```

Types: feat, fix, ci, refactor, test, docs, chore

## MR/PR Description

```
## Why
(Background / reason for change)

## How
(Implementation approach)

## What
(Changed files)
- `path/to/file` — description

## Verification
(How it was tested)
```

## RESUME.md Format

```markdown
## Current State
(Status of each project/branch)

## Next Steps
(Prioritized list)

## Key Info
(Important IDs, URLs, paths)
```
